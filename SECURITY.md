# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not open a public issue**.

Instead, report it privately by creating a **GitHub Security Advisory**:

1. Go to the repository's **Security** tab.
2. Click **Report a vulnerability**.
3. Provide a clear description, affected components, and reproduction steps.

If Security Advisories are unavailable for any reason, contact the maintainers
at [security@xteink.com](mailto:security@xteink.com).

## What to Include

Please include as much detail as possible:

- Firmware version / commit SHA
- Hardware revision (if known)
- Steps to reproduce
- Potential impact
- Any proof-of-concept details

## Disclosure Process

- We will acknowledge reports as quickly as possible.
- We will investigate, validate, and prioritize remediation.
- We may request additional details to reproduce and verify the issue.
- Once fixed, we will coordinate responsible disclosure and publish guidance.
